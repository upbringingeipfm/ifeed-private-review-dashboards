From raw signals to publishable intelligence.

This workflow defines how weekly research moves from tool prompts and pasted outputs into Obsidian notes, a raw-signal dashboard, the signal cockpit, and finally the public iFeed Weekly Signals issue.

1. Signal Capture

Tool-specific prompts

Maintain fixed prompts for Codex, ChatGPT, Claude, Perplexity, DeepSeek and Gemini, plus one miscellaneous prompt for newsletters, LinkedIn experts, regulators, companies and manual finds.

Seven-layer lens

Every scan should consider domain, domain lenses, functional areas, ecosystem area, governance, AI adoption and outward expression routing.

No early signal limit

Capture broadly first. Selection happens later. The raw layer is for awareness, blind-spot reduction and future traceability.

How it connects

The Markdown files are the capture ledger. Codex can parse them into a structured JSON corpus when available. Without Codex, the same files can be pasted into ChatGPT, Claude or Gemini for normalization using the same schema.

W25 test workspace

Use the signal management system to select a tool prompt, paste raw output, save in-browser, and export Markdown or JSON. The static dashboard cannot silently write to Obsidian, but it can generate the exact files that Codex or GitHub should preserve.

3. Refinement Into Weekly Dashboard

Normalize

Convert pasted outputs into one candidate register with title, date, source URL, domain, source type, tool origin, claim, relevance and confidence.

Verify

Check exact date, primary source, source-owner language, duplicate status, prior W19-W23 coverage and risk of overclaim.

Dashboard

Render every raw signal into a private review dashboard with filters, expanded cards, source audit, editorial action and selection controls.

4. Cockpit to iFeed Weekly Signals

Signal cockpit

The cockpit is the editorial decision room. It holds all raw signals, identifies the best 15-20, marks duplicates/context/drop items and preserves awareness signals that are not published.

Weekly digest

The public iFeed issue only imports the final nine. Each signal becomes a story with claim boundary, source posture, why it matters, Monday move and visual card logic.

5. Operating Without Codex

ChatGPT fallback

Paste the Markdown capture files and ask for normalization to the fixed JSON schema. Use smaller batches if subscription/context is limited.

No-subscription mode

Use one tool at a time, preserve raw Markdown, and ask only for narrow tasks: dedupe this batch, verify these ten sources, or score these signals.

Mobile mode

Capture links and notes into Obsidian, Notes, Google Docs or GitHub web editor. Full normalization can wait until laptop access.

6. CMS Boundaries

Fixed framework
  • Weekly folders and naming
  • Prompt library
  • Signal schema
  • Selection rubric
  • Source and claim policy
  • Digest card template
  • Netlify/GitHub deployment route
Incremental database
  • Raw signal corpus
  • Source ledger
  • Preliminary 20
  • Final 9 handoff
  • Prior coverage map
  • Reader lanes and domains
  • Rejected/noise archive
What the CMS must do

The user should edit only week dates, tool outputs, signal fields, selection decisions and public copy. The framework, schemas, file names, route structure, source policy and templates should stay fixed so each week can be repeated without breaking older weeks.

7. Sustainability Rules

Never overwrite working projects

Private review dashboards, signal cockpit and public weeklysignals stay as separate GitHub/Netlify projects.

Data before design

Verify sources, claims and placement before producing visual cards or public writing.

Every week is reproducible

A new week should be created by copying the week template, adding raw capture files, rendering a dashboard, selecting final nine and committing to GitHub.

8. What Still Needs To Be Built

Automation layer

A parser that reads the seven Markdown capture files, deduplicates signals and generates the weekly dashboard JSON automatically.

Handoff bridge

An export/import file that moves final nine signals from the private cockpit into the public iFeed Weekly Signals CMS without copy-paste drift.

Admin console

A simple interface for adding a new week, pasting raw outputs, editing source fields, choosing preliminary 20 and final nine, then exporting deploy-ready files.

QA gate

A checklist that blocks publish if source URLs, date windows, claim boundaries, duplicate checks, mobile view or prior-week coverage are incomplete.

Recommended long-term source of truth: GitHub repo + Markdown capture files + structured JSON corpus. Obsidian is the human writing layer; GitHub is the deployment and history layer.
